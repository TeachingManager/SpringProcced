지금까지는 스프링 빈을 등록할때 @Bean을 사용해 설정 정보에 직접 나열

컴포넌트 스캔 - 스프링은 설정 정보가 없어도 자동으로 스프링 빈 등록

@Autowired - 의존관계 자동 주입
생성자에 지정 자동으로 스프링 빈 주입
기본 조회 전략은 같은 타입의 빈



@ComponentScan

@Component가 붙은 모든 클래스를 스프링 빈으로 등록
스프링 빈의 이름은 클래스명, 맨 앞글자 소문자
@Component("이름") 으로 설정 가능
basePackages = "hello/core" 로 탐색 시작위치 지정 가능 - 모든 자바 코드를 보지 않도록
		{"",""}가능
		기본은 설정 정보 클래스의 패키지 - 설정정보를 프로젝트 최상단에 두기
스프링부트 사용시 대표 시작 정보인 @SpringBootApplication을 프로젝트 시작 위치에 둔다
				-(@ComponentScan을 포함)


컴포넌트 스캔에서 같은 빈 이름을 등록하면
자동 vs 자동
-Conflict 예외 발생
수동 vs 자동
-수동 빈 등록이 우선권 - 수동 빈이 자동 빈을 오버라이딩 (로그에 나옴)
--최근 스프링부트에서는 모두 오류가 발생하도록 기본값
